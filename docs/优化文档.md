# 监控告警中心数据源优化文档

## 目标

按照 `docs/api/m0-env-baseline.md` 最新数据库设计与约束，改造 Python API 与 Vue 页面，使“监控管理 -> 告警中心”相关菜单直接读写正确的 SQLite 表。

## 影响范围

- Python: `backend/app/routes/monitoring_target.py`
- Vue:
  - `frontend/src/api/monitoring.ts`
  - `frontend/src/views/monitoring/alert/index.vue`
  - `frontend/src/views/monitoring/alert/integration.vue`
  - `frontend/src/views/monitoring/alert/group.vue`
  - `frontend/src/views/monitoring/alert/inhibit.vue`
  - `frontend/src/views/monitoring/alert/silence.vue`
  - `frontend/src/views/monitoring/alert/notice.vue`
  - `frontend/src/router/index.ts`

## 步骤计划

1. 完成告警中心接口与数据库表映射设计（当前告警、历史告警、告警配置、集成/分组/抑制/静默/通知）。
2. 改造 Python 路由，告警中心接口从 Manager 代理改为 SQLite 直连 ORM（遵循新约束）。
3. 改造 Vue API 类型与各告警页面字段映射，确保页面展示与提交字段对应新表。
4. 联调与回归：执行后端单测与前端类型检查（可执行范围内），修正兼容问题。
5. 输出结果与遗留项。

## 表映射（实施口径）

- `/monitoring/alerts/current` -> `single_alerts`
- `/monitoring/alerts/history` -> `alert_history`
- `/monitoring/alert-rules*` -> `alert_defines`
- `/monitoring/alert-integrations*` -> `notice_receivers`
- `/monitoring/alert-groups*` -> `alert_groups`
- `/monitoring/alert-inhibits*` -> `alert_inhibits`
- `/monitoring/alert-silences*` -> `alert_silences`
- `/monitoring/alert-notices*` -> `notice_rules`
