# UI Layout Contract: Frontend Vben UI Refactor

## Purpose

定义本次前端重构必须满足的布局契约、主题契约和受保护逻辑边界。该契约只约束前端 UI 层，不涉及后端 API 变更。

## Layout Contract

1. `BasicLayout` 必须占满视口高度。
2. 侧边栏和头部必须固定在应用骨架中，不跟随主内容滚动。
3. 内容区必须是唯一的主滚动容器。
4. 不允许出现应用级双滚动条。
5. 侧边栏折叠、展开和响应式收缩时，Logo、菜单图标和标题必须保持可用。

## Navigation Contract

1. 所有现有一级菜单、子菜单和动态自定义视图必须继续存在。
2. `selectedKeys` 和 `openKeys` 的更新行为必须与路由同步。
3. 新增面包屑只能消费已有路由状态，不能引入新的路由依赖。
4. 不允许改动 `hasPermission` 和 `hasAnyPermission` 的判断语义。

## Theme Contract

1. 必须同时支持 `light` 和 `dark` 两个主题模式。
2. 亮色主题以白色商务风格为主：白底、细边框、低阴影、蓝色激活态。
3. 暗色主题必须保持文本、边框、交互态可读，不要求完全复刻亮色视觉。
4. 主题切换必须继续使用现有状态来源，不新增破坏性状态管理。

## Protected Logic Contract

以下逻辑仅允许做样式承载层调整，不允许变更功能语义：

- 站点配置加载
- 自定义视图加载
- 通知模块初始化
- 通知 WebSocket 建连
- 路由监听与菜单同步
- 用户信息读取
- 语言切换
- 权限判断

## Page Surface Contract

1. 页面容器必须支持统一的标题区、工具栏和内容块节奏。
2. 列表页应统一筛选栏、操作区、表格容器和分页容器。
3. 详情页/配置页应统一信息区、表单区和操作区。
4. 仪表盘应统一指标卡、图表面板和辅助信息面板的边框与间距规范。
